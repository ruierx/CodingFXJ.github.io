v-if 和v-for优先级

v-for 优先级高于v-if （从源码中可知）

vue组件data为什么是函数

vue根组件是单例的，只有一个

vue其他组件可能有多个实例，为了保障各组件数据不互相干扰，

如果是对象，这样多个组件会共享数据

computed和watch

computed：依赖vm中data的属性变化而变化，函数必须用return返回。不要对data中的属性进行赋值操作，会形成死循环。

模板页面需要处理复杂的逻辑运算时，可以处理同步数据，不能处理异步数据。

使用场景：当一个值受多个属性影响的时候——————购物车商品结算

watch：函数是不需要调用的，函数名称必须要和data中的属性名一致。

可以同步、异步处理数据，适用于监听数据与事件交互，当数据发生改变时触发多个事件。

使用场景：当一条数据的更改影响到多条数据的时候————-搜索数据

区别：

DIFF算法

将虚拟dom变化的地方高效的更新在真实dom上，时间复杂度O(n)

diff执行过程是深度优先、同级比较的策略

vue设计原则理解

渐进式js框架（学习曲线是循序渐进的），核心库只关注视图层，易用、灵活、高效

灵活性：可以按需引入自己所需的库

高效性：超快的虚拟dom和diff算法

MVVM

dom监听model数据，数据绑定到view，实现数据双向绑定。

实现了前后端数据的通信，低耦合

Vue要求模板只能有一个根节点原因

从三方面考虑：

new Vue({el:’#app’}) ,实例化vue，需要开启一个入口

树状数据结构，需要一个根作为遍历的起点。

diff算法要求

组件化

一个页面中所有的处理逻辑全部放在一起，那么处理起来会非常复杂混乱，不利于后续的管理以及扩展。

把一些独立的功能单独抽取出来形成一些小的、独立的、可复用的模块，便于解耦、管理和维护，提升开发效率，测试性，复用性。

组件分类：页面组件、业务组件、通用组件

前端性能优化

前端性能：从网站生成到代码开始运行消耗浏览器、服务器的资源统称为性能。

CSS层：将 CSS 放在文件头部、降低 CSS 选择器的复杂性、使用 flexbox布局，正确使用display，不滥用float，不过多声名font-size，值为0时不需要单位，避免使用行内样式

JS层：JavaScript 文件放在底部、使用事件委托、优先选择原生方法，避免频繁改变DOM元素样式，精炼JS代码，选择时间复杂度和空间复杂度较好的方案，使用节流

静态资源：使用字体图标 iconfont 代替图片图标、图片懒加载 ，响应式加载 ，图片压缩、尽可能利用 CSS3 效果代替图片

网络层： 使用防抖函数减少 HTTP 请求、使用 HTTP2、

服务器：启用Gzip压缩，静态资源使用 CDN托管加速，减少cookie头信息大小

浏览器：善用缓存，不重复加载相同的资源、减少重绘重排

VUE层：按需加载代码，提取第三库代码、

Vue优化

  1. 路由懒加载

  2. keep-alive缓存页面

  3. 使用v-show服用DOM

  4. v-for遍历避免同时使用v-if，使用computed提前处理数据

  5. 图片懒加载

  6. 提取组件的 CSS 到单独到文件

  7. 第三方插件按需导入（element-ui）

  8. 利用Object.freeze()提升性能

  9. 扁平化 Store 数据结构

  10. 手动销毁vue本身组件以外的事件（比如定时器）

  11. 无状态的组件标记为函数式组件<template functional> </template>

  12. 子组件分割，computed中需频繁访问的数据，可定义变量保存，避免this.调用

Vue3.0新特性

更快：虚拟DOM重写、优化slot的生成

更小：核心库体积更小

nextTick实现原理

它可以在DOM更新完毕之后执行一个回调

数据双向绑定（响应式原理）

概念：数据变化更新视图、视图变化更新数据

设计思想：观察者模式

核心是 Object.defineProperty()方法进行数据劫持

vue3.0版本用的是代理对象（new Proxy()）来做响应式

实现步骤：

1.实现一个监听者Oberver来劫持并监听所有的属性，一旦有属性发生变化就通知订阅者

2.实现一个订阅者watcher来接受属性变化的通知并执行相应的方法，从而更新视图

3.实现一个解析器compile，可以扫描和解析每个节点的相关指令，并根据初始化模板数据以及初始化相对应的订阅者

递归组件

概念：可以在它们自己的模板中调用自身的，一定要有一个结束的条件。

应用：树状多级菜单
